samcat(1)

NAME

samcat — concatenate and print SAM and BAM files

SYNOPSIS

samcat [-bnv] [-f FLAGS] [-o FILE] [-O FORMAT] [FILE]...

DESCRIPTION

The samcat utility reads files in SAM or BAM format, merges their headers, and writes the merged headers and alignment records, sequentially from each specified file in turn, to standard output or to a specified file. If no input files are specified, or when FILE is a single hyphen ("-"), samcat reads from standard input.

The following options are accepted:

-b
Write output in BAM format.
-f FLAGS
Display only alignment records matching FLAGS, as described below. This option may be used more than once, with all FLAGS given accumulating.
-n
Suppress headers from the output.
-o FILE
Write to FILE rather than standard output.
-O FORMAT
Write output according to FORMAT, as described below.
-v
Display file information and statistics, on standard error.

Filtering alignment records

The -f FLAGS option lists combinations of flags to be included and/or excluded in the format [[+]inc][-excl] where either clause may be omitted, and the "+" introducing the inc clause is also optional. Each clause is either numeric or a set of the flag characters listed below.

Each alignment record will be emitted only if it has all the flags listed in inc set, and all of the flags in excl not set.

Output formats

By default, output is written as text in SAM format, with the FLAG field displayed in decimal. The following alternatives can be selected:

bam
Compressed binary BAM format.
hex
Text SAM format, with the FLAG field displayed in hexadecimal.
text
Text SAM format, with the FLAG field displayed symbolically as a set of characters representing individual bit flags:
uRead is unmapped
f/rMapped to the forward (reverse) strand
UMate is unmapped
F/RMate is mapped to the forward (reverse) strand
pPaired (lowercase "p")
PProperly paired (capital "P")
1/2Read is the first (second) in a pair
sRecord is not a primary alignment
qFailed platform or vendor quality checks
dPCR or optical duplicate
Note that this differs slightly from samtools's symbolic flag notation, which uses "f" for quality failure and doesn't explicitly denote that reads are on the forward strand.
ubam
Uncompressed binary BAM format. Sets the compression level so as to maximise writing speed, at the cost of increased output size.
