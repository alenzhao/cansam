samsplit(1)

NAME

samsplit — split a SAM or BAM file into separate read groups

SYNOPSIS

samsplit [-b] [-f FLAGS] [-o FILE] [-q NUM] [-z NUM] FILE [TEMPLATE]

DESCRIPTION

The samsplit utility reads one file in SAM or BAM format, and breaks it up into several output files, one for each read group present in the input. When FILE is a single hyphen ("-") or if it is absent, samsplit reads from standard input.

The following options are accepted:

-b
Write output files in BAM format.
-f FLAGS
Emit only alignment records matching FLAGS, which lists flags to be included and/or excluded in the same [[+]inc][-excl] format used by samcat(1). This option may be used more than once, with all flags given accumulating.
-o FILE
Write all selected alignment records to FILE, as well as writing them to the appropriate read group output file.
-q NUM
Discard alignment records with mapping quality less than NUM.
-z NUM
Set output file compression level to NUM.

For each read group in the input, headers and those alignment records within that read group that are selected by the various filtering options are written to a file named according to TEMPLATE, within which the following escape sequences are expanded:

XY
The value of the read group header's XY field.
%#
The index (starting from 1) of the read group header within all the headers.
*
The input FILE basename, i.e., FILE with the extension and any leading directory part removed.
%.
Either "sam" or "bam", as appropriate for use as the extension for the chosen output format.
%%
A single "%" character.

If the TEMPLATE argument is not specified, read group files are named according to the template "<b>%*-%ID.%.</b>".

The -o FILE filename is expanded similarly, though XY and %# are not useful here as no particular read group is associated with this output file.

LIMITATIONS

At present, all output files are opened simultaneously, so the number of read groups that a file can be split into is limited by the process limit on open file descriptors.
