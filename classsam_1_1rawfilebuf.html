Detailed Description

Provides unbuffered bulk access to a Unix-style file descriptor. The usual public std::streambuf methods are available as follows:

  • sgetn() corresponds directly to a read(2) system call;
  • sputn() corresponds to a write(2) system call, invoking it repeatedly if necessary until all data has been written;
  • pubseekoff() and pubseekpos() correspond directly to lseek(2) system calls;
  • in_avail() corresponds (for ordinary files) to a combination of lseek(2) and fstat(2) system calls;
  • methods for character-orientated input/output (sgetc(), sputc(), etc) are not implemented and simply throw std::logic_error;
  • other methods (e.g., pubimbue()) have no effect.

These methods retry their system calls if they are interrupted by signal delivery. Thus calling code does not need to deal with EINTR or foreshortened interrupted writes itself. If a system call fails for other reasons, these methods throw sam::system_error exceptions accordingly.

(To be precise, this class overrides the seekoff(), seekpos(), showmanyc(), xsgetn(), and xsputn() protected methods, and trivially overrides overflow(), uflow(), and underflow() as throw statements; all the others are inherited as no-ops from std::streambuf.)
